**BrewOps** is a modern, high-performance **Multi-Tenant SaaS (Software as a Service)** platform built specifically for specialty coffee shops and cafe businesses. It provides owners, managers, and baristas with a unified cloud interface to manage point-of-sale transactions, automate inventory tracking, schedule staff attendance, and forecast sales trends using built-in AI models.

---

## 🚀 Key Features

*   **🛒 POS Terminal:** A clean, lightning-fast cash-register catalog with product modifiers, shopping cart, customizable taxes, and split payment methods (Cash, Credit Card, Digital Wallets, QRIS).
*   **📦 Inventory & Stock Control:** Real-time stock level deduction per sales order based on custom recipes, critical stock alerts, manual adjustments, and vendor management.
*   **👥 Staff Management:** Barista attendance tracking with simple Clock-In/Clock-Out and automated work hour calculations.
*   **🔮 AI Sales Predictions:** Sales forecasts for the next 7 days and inventory run-out estimations generated by a lightweight predictive analytics engine.
*   **📱 Online QR Menu:** Self-ordering public web catalog for customers to scan table QR codes, queue up orders directly to the kitchen, and pay at the counter.
*   **📊 Reports & Export:** Detailed financial sales analytics, hourly sales trends, best-selling product distributions, and CSV export.
*   **⚙️ Tenant Settings:** Customizable cafe profile, active payment gateways configuration (Midtrans, Stripe, Xendit), tax rates, and active currency adjustments.

## 📦 Getting Started

### Prerequisites

*   Node.js (v18.x or v20.x recommended)
*   npm or yarn
*   PostgreSQL Database (Local or cloud-hosted instance, e.g., Supabase / Neon)
*   *Alternatively:* Docker & Docker Compose (for quick local database instantiation)

### Installation & Setup

1.  **Clone the repository:**
    ```bash
    git clone https://github.com/your-username/brewops-coffeeshop.git
    cd brewops-coffeeshop
    ```

2.  **Install dependencies:**
    ```bash
    npm install
    ```

3.  **Configure environment variables:**
    Copy `.env.example` to create a local `.env` configuration file:
    ```bash
    cp .env.example .env
    ```
    Open `.env` and fill in your connection secrets:
    *   Set a secure `JWT_SECRET` (at least 32 characters long).
    *   Set `DATABASE_URL` pointing to your PostgreSQL instance.

4.  **Launch local PostgreSQL database (Optional - via Docker):**
    If you do not have a PostgreSQL database ready, run:
    ```bash
    docker-compose up -d
    ```

5.  **Run Database Migrations & Seeds:**
    Push the database schema and seed the initial mock data (owner, manager, staff credentials, and catalog):
    ```bash
    npx prisma migrate dev --name init
    npx prisma db seed
    ```

6.  **Run the local development server:**
    ```bash
    npm run dev
    ```
    Open [http://localhost:3000](http://localhost:3000) in your browser.

## 🔒 Security & Middleware Protections

*   **Production JWT Safety:** In production (`NODE_ENV=production`), the application forces a secure `JWT_SECRET` check and rejects default development secrets to prevent token tampering.
*   **Route Guards:** Next.js middleware (`src/middleware.ts`) automatically intercepts incoming requests to `/dashboard/:path*`, verify session cookies, and redirect unauthenticated users to `/login`.
*   **Database Fallback Policy:** The local development in-memory data fallback automatically turns OFF in production. The app will never leak mock data or bypass database connection failures when deployed.
